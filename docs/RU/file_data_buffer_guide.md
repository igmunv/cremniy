# Работа с FileDataBuffer

`FileDataBuffer` — это ядро системы управления данными в Cremniy. Он реализует паттерн "Единый источник правды" для всех операций с содержимым файлов.

---

## Основная концепция

Вместо того чтобы каждая вкладка хранила свою копию данных или сама читала файл, все вкладки одного файла разделяют один экземпляр `FileDataBuffer`.

**Преимущества:**
1.  **Мгновенная синхронизация**: Изменение байта в Hex-редакторе тут же отображается в текстовом редакторе.
2.  **Экономия памяти**: Большие файлы не дублируются в памяти.
3.  **Безопасное сохранение**: Только буфер знает, как записывать данные на диск, что исключает потерю данных при конфликтах вкладок.

---

## Использование во вкладках (ToolTab)

Каждая вкладка получает указатель на `m_dataBuffer` в конструкторе.

### 1. Чтение данных
Для отображения данных используйте методы буфера:
```cpp
QByteArray data = m_dataBuffer->read(offset, length);
char byte = m_dataBuffer->at(pos);
```

### 2. Редактирование данных
Никогда не меняйте локальные копии! Используйте методы буфера:
```cpp
m_dataBuffer->replace(pos, byte); // Для одиночного байта
m_dataBuffer->replace(pos, length, newData); // Для диапазона
```

### 3. Реактивное обновление
Вкладки должны реагировать на сигналы буфера, переопределяя методы `ToolTab`:
*   `onByteChanged(qint64 pos)` — изменился один байт.
*   `onBytesChanged(qint64 pos, qint64 length)` — изменился диапазон.
*   `onDataChanged()` — файл был полностью перезагружен.

---

## Жизненный цикл сохранения

В новой архитектуре **вкладки не реализуют сохранение**.

1.  Пользователь нажимает `Ctrl+S` или `File -> Save`.
2.  `IDEWindow` посылает сигнал в `FilesTabWidget`.
3.  `ToolsTabWidget` вызывает:
    ```cpp
    m_sharedBuffer->saveToFile(m_filePath);
    ```
4.  Буфер сохраняет данные на диск, сбрасывает флаг модификации и уведомляет все вкладки, чтобы они убрали "звездочку" (индикатор изменений).

---

## Работа с большими файлами

`FileDataBuffer` поддерживает эффективную работу с файлами любого размера:
*   Используйте `isLargeFile()`, чтобы определить, нужно ли отключать тяжелые функции (подсветку синтаксиса, word wrap).
*   Для текстовых редакторов рекомендуется использовать замену `TAB` на пробелы и ограничение длины строки для повышения производительности.
